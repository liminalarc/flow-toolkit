## Problem
The toolkit is explained entirely in prose across README + CLAUDE.md. Newcomers
(and the maintainer running it across 4-5 projects) have no visual model of how the
pieces fit — the checkpoint loop, the index/detail spec model, which hooks fire on
which events and how they all funnel through `flow-preflight.sh`, and how install
distributes to every profile. "How all of this works" lives only in the reader's head.
This is a documentation upgrade: cleanup + pictures, no new automation.

## Value
As a developer evaluating or onboarding to flow-toolkit, I want diagrams of the
core systems so that I can grasp the workflow, spec model, hook architecture, and
install flow at a glance instead of reconstructing them from prose.

## Scope
**In:**
- New `docs/ARCHITECTURE.md` — a how-it-works doc holding four Mermaid diagrams,
  each with a short prose caption tying it to the source files it depicts.
- Diagram 1 — **Dev cycle + checkpoint loop**: Understand → Plan → Checkpoint
  (sign-off gate) → Build (test-first, per-slice commits) → Done, with the
  deferral-protocol branch and the DONE-gate.
- Diagram 2 — **Spec model + lifecycle**: index + detail-file split, flat vs
  directory (orchestrator + task files) shape, status vocabulary, active → DONE →
  archive flow.
- Diagram 3 — **Hook / event architecture**: Claude Code events → each hook, and
  guards + `/flow-lint` + `/flow-ship` all calling the single-source
  `flow-preflight.sh` (git-state / resolved / wellformed).
- Diagram 4 — **Install / distribution**: git clone → installer → per-profile
  detection → force-copy commands + hooks + additive settings.json merge →
  update via git pull + reinstall.
- Embed Diagram 1 (dev cycle) in `README.md` with a link to the full set.
- Link `docs/ARCHITECTURE.md` from README (nav/top) and CLAUDE.md "See Also".
- Add `docs/` to the Project Structure tree in root CLAUDE.md.
- Light prose cleanup encountered while adding diagrams (fix any stale reference).

**Out:**
- No rendering pipeline, image generation, or CI diagram-lint — plain-text Mermaid
  renders natively on GitHub; keeping diagrams accurate is a manual done-check, not
  a new mechanism (thin slices, no premature abstraction).
- No per-command sequence diagrams beyond the four above (add later if a real need appears).
