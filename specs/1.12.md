## Problem
The toolkit is explained entirely in prose in a 742-line README plus CLAUDE.md.
It is both too long to be a front door and missing the things a newcomer most needs:
a visual model of how the pieces fit (the checkpoint loop, the spec model, which
hooks fire on which events and funnel through `flow-preflight.sh`, how install
distributes per-profile) and — critically — **when the sub-agents get dispatched**
(run→implementer/verifier, hunt→researchers, review→reviewers, pr→pr-reviewers).
"How all of this works, and how I use each piece" lives only in the reader's head.
This is a documentation upgrade: slim the README to a front door, and give the
reference depth a real home — diagrams + a usage guide with worked examples.

## Value
As a developer evaluating or onboarding to flow-toolkit, I want a lean README plus
a complete reference — systems diagrams (including agent dispatch) and a usage guide
that covers every command/skill/agent/hook with examples — so I can grasp the toolkit
at a glance and go deep on demand, instead of scrolling a 742-line README.

## Scope
**In:**
- Slim `README.md` to a front door (~180 lines): what it is, install, a clear
  **pre-plugin manual-install uninstall/migration note**, Quick Start, the embedded
  dev-cycle diagram, a compact "what's in the box" table, links to the two docs, Updating.
- `docs/ARCHITECTURE.md` — five captioned Mermaid diagrams, each naming its source files:
  1. **Dev cycle + checkpoint loop** — Understand→Plan→Checkpoint→Build→Done, deferral branch, DONE-gate.
  2. **Spec model + lifecycle** — index+detail split, flat vs directory, status vocabulary, active→DONE→archive.
  3. **Hook / event architecture** — Claude Code events → each hook, all funnelling through `flow-preflight.sh`.
  4. **Install / distribution** — plugin + fallback installer, per-profile detection.
  5. **Agent dispatch** — which skill fires which sub-agent(s), and at what moment (the "when agents fire" picture).
- `docs/GUIDE.md` — the comprehensive usage manual: concepts (spec model, CLAUDE.md
  hierarchy, config backend, autonomy); every skill & command with what/how/example
  and **which agents it dispatches and when**; the 5-agent catalog; hooks + preflight;
  worked walkthroughs (full spec lifecycle, cross-cutting spec with implementer/verifier
  fan-out, a `hunt` with researcher fan-out); customizing line caps; project-specific commands.
- Embed the dev-cycle diagram in `README.md` and link both docs from it.
- Link `docs/ARCHITECTURE.md` + `docs/GUIDE.md` from CLAUDE.md "See Also"; add `docs/` to Project Structure.
- Fix stale references encountered (bare `/flow-lint`, `commands/flow.md`, etc.).

**Out:**
- No rendering pipeline, image generation, or CI diagram-lint — Mermaid renders natively
  on GitHub; diagram accuracy is a manual done-check, not a new mechanism.
