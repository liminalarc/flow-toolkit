## Problem
`flow` (the `--add` template, `--ideas` lenses, `--clean` rules) and `flow-hunt` (persona panel × dimensions) are the remaining heavy commands that load whole. Once the skill + fan-out pattern is proven on `flow-review`/`flow-pr` (1.8/1.9), these migrate to reclaim their progressive-disclosure and fan-out wins.

## Value
As a flow user I want `/flow` to load only the path I invoke (implement vs `--add` vs `--ideas` vs `--clean`) and `/flow-hunt` to fan its persona panel out to researcher agents, so that the two most-used commands stay lean and hunts run in parallel.

## Scope
**In:** convert `flow` → skill (paths + templates/lenses/rules as `reference/*`); convert `flow-hunt` → skill with researcher sub-agents (persona × dimension fan-out); entry points stay `/flow`, `/flow-hunt`.
**Out:** thin/deterministic commands (`flow-lint`, `flow-ship`) stay commands (1.2 D-b); plugin packaging (1.10).

## Acceptance criteria
- [ ] `flow` is a skill; each path's detail is a `reference/*` loaded only on that path.
- [ ] `flow-hunt` is a skill fanning out one researcher agent per persona × dimension; synthesized.
- [ ] Both entry points (`/flow`, `/flow-hunt`) unchanged; no behavior regression.
- [ ] README + docs updated.

## Plan (thin slices)
1. [ ] `flow` path split into `reference/*` + `SKILL.md`.
2. [ ] `flow-hunt` skill + researcher agent fan-out.
3. [ ] Smoke-test both; update docs.
