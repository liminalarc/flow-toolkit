## Problem
`flow` (implement + `--add`/`--ideas`/`--clean`/`--condense`) and `flow-hunt` (persona panel × dimensions) are the last heavy commands that load whole. With the skill + fan-out pattern proven on `flow-review`/`flow-pr` (1.8/1.9), these migrate to reclaim their progressive-disclosure and fan-out wins.

## Value
As a flow user I want `/flow` to load only the path I invoke (implement vs `--add` vs `--ideas` vs `--clean` vs `--condense`) and `/flow-hunt` to fan its research out to parallel agents, so that the two most-used commands stay lean and hunts run in parallel.

## Scope
**In:** `flow` → router skill (`SKILL.md` dispatches; substantial paths + shared authoring templates as `reference/*`); `flow-hunt` → fan-out skill with a new read-only `flow-researcher` agent (per-dimension, persona-aware, both offline and `--deep`); delete both old command files; entry points stay `/flow`, `/flow-hunt`.
**Out:** thin/deterministic commands `flow-lint`/`flow-ship` stay commands (1.2 D-b); plugin packaging (1.10).

## Acceptance criteria
- [ ] `flow` is a skill: `SKILL.md` routes by invocation; each substantial path (`implement`/`add`/`clean`/`condense`) is a `reference/*` loaded only on that path; shared authoring templates + terseness live in one `reference/authoring.md`; trivial paths (no-arg, `--ideas`) stay inline. — no behavior regression vs `commands/flow.md`.
- [ ] `flow-hunt` is a skill fanning out one `flow-researcher` per research dimension (persona-aware), in **both** offline and `--deep` modes; main thread synthesizes the scored opportunity report.
- [ ] New read-only `flow-researcher` agent (Read/Grep/Glob/WebFetch/WebSearch) mirroring `flow-reviewer`'s boundaries (researches + reports, never edits).
- [ ] Both entry points unchanged; `commands/flow.md` + `commands/flow-hunt.md` deleted; skill outranks any stale installed command (1.8 precedent).
- [ ] Installers dry-run clean on both profiles (skills + agents land, no stale command); README `/flow` + `/flow-hunt` sections + command table + CLAUDE.md structure updated.

## Decisions
- 2026-07-14 — flow-hunt fan-out is **per research dimension (persona-aware)**, not the seed's literal persona×dimension matrix (12-30 redundant agents); refines AC wording, mirrors flow-review/flow-pr fan-out shape.
- 2026-07-14 — Fan-out applies to **both** offline and `--deep`; the `--deep` flag only toggles whether each researcher web-searches. One architecture, no dual path.
- 2026-07-14 — **flow-hunt carries no `reference/` files**: unlike flow-review/flow-pr (static per-lens rubrics), its dimensions are derived per-project at Phase 0, so the researcher's method lives in the agent definition + dispatch context.
- 2026-07-14 — `flow` router keeps big templates **out** of always-loaded `SKILL.md` (the progressive-disclosure win); DRY → shared authoring in one `reference/authoring.md`.
- 2026-07-14 — Run authorized `auto-build`-style overnight (user grant); verifier **blocking** per slice, stop-and-document on unresolved FAIL.
