## Problem
1.1 makes *new* specs terse and *warns* when a spec drifts over budget, but existing specs across live projects predate those rules and can only be fixed by hand. And the soft budget is a line count — it can't see cross-section restatement or prose that should be bullets. A condense pass (one spec, a whole `specs/` dir, or a report-only audit) closes both gaps.

## Value
As a developer running flow across several existing projects, I want a command that rewrites bloated specs to the terseness rules — or just reports where they fall short — so that I can migrate a backlog to the standard and keep it there, without hand-editing or losing detail.

## Scope
**In:** `/flow --condense [<id> | --all] [--check]`.
- `<id>` condenses one spec; `--all` sweeps every `specs/<id>.md` (project-wide migration).
- Rewrite applies 1.1's rules losslessly (one job per section, no cross-section restatement, prose→bullets); **copies the Progress log verbatim** and preserves every AC's substance; shows a diff and confirms **per spec** before any write; echoes a no-op when already terse; amends **freely regardless of index status**.
- `--check` is report-only: audits against the *full* terseness rules (budget **and** the qualitative restatement/repetition rules a line count can't catch) and reports findings without proposing a rewrite — usable on already-condensed specs as ongoing hygiene.

**Out:** format migration (`/flow-lint --migrate` owns legacy inline → index model); the terseness rules + soft budget warning themselves (1.1); interactive field-level editing (`/flow <id>` resume covers ad-hoc edits).

## Acceptance criteria
- [ ] Invocable as `/flow --condense <id>` and `/flow --condense --all`; echoes a no-op when a spec is already under budget + non-repetitive.
- [ ] `--check` reports over-budget **and** restatement/repetition findings without writing; returns a clean report on an already-terse spec.
- [ ] Diff shown and confirmed **per spec** before any write — never rewrites silently, even under `--all`.
- [ ] Lossless: Progress-log entries reproduced verbatim; every AC's substance preserved (verified on a real bloated spec).
- [ ] No status special-casing — a DONE/archived spec condenses the same as any other.
- [ ] README + `/flow-lint --specs` cross-reference the command; README additions stay lean.

## Plan (thin slices)
1. [ ] Author `/flow --condense [<id>|--all] [--check]` in `flow.md` — rewrite rules, verbatim-Progress-log guarantee, per-spec diff+confirm, no-op echo, free-amend, and the `--check` report mode.
2. [ ] Cross-reference: `/flow-lint --specs` points over-budget specs at `/flow --condense`; document in README (spec-model + `/flow` sections), lean.
3. [ ] Smoke-test on a real bloated scratch spec: rewrite is lossless (Progress log verbatim, ACs intact) + under budget after; `--check` reports without writing.

## Decisions
- 2026-07-14 — Broadened from the stub's single-spec condense to include **project-wide `--all`** and a **report-only `--check`** mode, per real multi-project use.
- 2026-07-14 — Invocation `/flow --condense`, **not** `/flow --clean` or `/flow-lint --fix`: those are *mechanical*; terseness rewrite/audit is a *judgment* pass. Lint's `--specs` stays the mechanical budget INFO; `--condense --check` is the qualitative audit (restatement/repetition) a line count can't catch.
- 2026-07-14 — **Amend freely** (no DONE/archived gate) — diff+confirm is the safeguard.
- 2026-07-14 — Runs **inline**, not via a spec-writer sub-agent — keeps 1.3 independent of 1.2.
- Boundary: `/flow-lint --migrate` = *format* migration; 1.3 = *content-quality* migration + audit. Distinct.
