## Problem
1.1 makes *new* specs terse and *warns* when any spec drifts over budget, but offers no way to fix a bloated existing spec — you trim it by hand. A command that condenses a detail file (losslessly) closes that loop.

## Value
As a developer with a bloated spec, I want a command that rewrites it to the terseness rules so that I can bring it under budget without hand-editing or dropping detail.

## Scope
**In:** a `/flow`-driven condense pass over one `specs/<id>.md` — apply 1.1's rules (one job per section, no restatement, prose→bullets, append-only Progress log), preserve every acceptance detail and the Progress-log history, show a diff before writing.
**Out:** index normalization (that's `/flow --clean`); the terseness rules + soft warning themselves (owned by 1.1).

## Acceptance criteria
- [ ] Invocable against one spec id; refuses/echoes a no-op when already terse.
- [ ] Diff shown and confirmed before any write — never rewrites silently.
- [ ] No acceptance detail or Progress-log entry lost (verified on a real bloated spec).
