## Problem
Every existing flow agent is static ‚Äî `flow-verifier`/`flow-pr-reviewer` read a diff, `flow-reviewer`
reads source. **None runs the app.** A class of interface defects only surfaces when you *use* it: does
a screen match the design system, and can a user finish the intended flow without friction. This is the
**Validate** step of engineer-directed delivery pointed at the interface ‚Äî the **live-driving complement**
to `review`'s static UX lens.

## Value
As an engineer directing agentic delivery I want an agent that drives the running app and scores its UI/UX
against a rubric, so interface regressions and flow friction are caught in the Validate step ‚Äî under my
gate ‚Äî not by me clicking around after the fact.

## Scope
**In:**
- **`flow-ux-validator`** agent (read-only re project files; `Read/Grep/Glob/Bash`) ‚Äî the reusable primitive.
  Per run: applicability-check ‚Üí resolve rubric ‚Üí drive app (Playwright-first, vision fallback) ‚Üí capture
  screens to scratch ‚Üí score **ONE** lens ‚Üí prioritized findings. Never edits/fixes project files.
- **`/flow:validate <screen|flow> --intent "<outcome>" [--ui|--ux] [--design-system <path|url>]`** ‚Äî thin
  launcher: pick lens(es), dispatch the agent (no flag ‚áí both, serialized), synthesize one assessment. The
  human entry point into UI/UX review.
- Two baseline rubrics ‚Äî `reference/ui.md` (design-system conformance) + `reference/ux.md` (Nielsen
  heuristics + WCAG + intent/friction) ‚Äî and `reference/driving.md` (how to drive; loaded only on that path).
- **Applicability-aware:** non-UI repo (no runnable UI / run cmd / driver) ‚áí clean `not applicable: <reason>`
  exit ‚Äî never a hallucinated critique.
- Docs: README, `docs/guide.md`, `docs/how-it-works.md` (dispatch diagram), `docs/architecture.md` (ADR:
  the live-driving agent), root `CLAUDE.md` catalog + `/flow:validate` usage examples.

**Out:**
- `/flow:run` **done-gate** integration ‚Üí **1.15** (depends on 1.14 ‚Äî the gate needs the agent first).
- **Persisted / self-maintaining project rubric** (`.flow/validate/*.md`, bootstrap + drift-refresh) ‚Üí
  **1.16**. 1.14 takes project specifics as per-run args (`--design-system`/`--intent`, or infer from source).
- Closed-loop auto-fix (agent validates + reports; the orchestrator/engineer owns any fix loop).
- PR-review caller (`/flow:pr`) ‚Äî judged overkill; deliberately not specced.
- Replacing `review`'s static UX lens; a visual-regression baseline system (reuse the target's if present).

## Decisions
- 2026-07-18 ‚Äî **Agent vs skill.** The agent is the reusable primitive; the skill is a thin launcher.
  Applicability-check + rubric-resolution live in the **agent** so every caller (validate skill now,
  `/flow:run` in 1.15) reuses identical behavior ‚Äî skills don't call skills.
- 2026-07-18 ‚Äî **Named `flow-ux-validator`, not "validator".** Scoped to UI/UX, distinct from the static
  `flow-verifier`/`flow-pr-reviewer` (which own correctness + clean code). This is the only agent that
  drives the live app.
- 2026-07-18 ‚Äî **Rubric = layered.** Baseline heuristics ship in `skills/validate/reference/{ui,ux}.md`;
  project specifics (design system, intent) are explicit input. 1.14 passes them as per-run args (or infers
  from source); a **persisted, self-maintaining project rubric** (`.flow/validate/*.md`) is **1.16**, read by
  both the standalone skill and 1.15's gate.
- 2026-07-18 ‚Äî **Driving.** Playwright-first (deterministic, via Bash); computer-use/vision documented
  fallback. Applicability-aware so non-UI repos exit N/A.
- 2026-07-18 ‚Äî **Lifecycle gate split to 1.15** (the gate needs the agent first). PR-review caller judged
  overkill; not specced.

button (`#ff2d95`, `5px 7px`, `2px` radius), placeholder-only inputs (no `<label>`), `#cccccc` sub-contrast
hint, ambiguous "Go" label. Drove it for real with **Playwright** (login‚Üídashboard, 2 screenshots). Then ran
three `flow-ux-validator` stand-ins (via `general-purpose`, since name-dispatch needs a restart):
- **UI lens** ‚Äî caught all button token breaks (CRITICAL), unlabeled inputs + sub-contrast hint (HIGH),
  ambiguous label (LOW); bonus off-scale margin; noted only the default state was captured. Grounded, in-lens.
- **UX lens** ‚Äî intent achieved (no critical); WCAG 3.3.2/1.3.1/4.1.2 + 1.4.3 findings; left button *styling*
  to the UI lens; abstained on undriven states.
- **N/A path** ‚Äî pointed at flow-toolkit (no app) ‚áí clean `NOT APPLICABLE`, zero fabricated findings.

**Manual, post-restart (for the user):** confirm `/flow:validate` + the `flow-ux-validator` agent dispatch by
