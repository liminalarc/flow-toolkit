## Problem
v1.0.0 shipped the `flow` plugin, but two gaps only bite a *pure* plugin install (no leftover installer files) — currently masked by the manual-install artifacts still on dev machines:
1. **Agent dispatch** — skills dispatch agents by **bare** name (`flow-implementer`), but plugin agents resolve **only by scoped name** (`flow:flow-implementer`); bare works today only because leftover `~/.claude*/agents/flow-*.md` shadow the plugin (confirmed via Claude Code docs).
2. **`flow-preflight.sh` discovery** — `/flow:ship` & `/flow:lint` search *profile* hook dirs; the plugin ships the script in its cache, so a pure-plugin user gets "reinstall flow-toolkit."

Also: going plugin-only needs a repeatable **purge** for anyone who used the old manual install (`claude plugin uninstall` only clears the cache; old files + `settings.json` hooks are manual).

## Value
As a flow user who installs *only* the plugin, I want agent-dispatching skills and `/flow:ship`/`/flow:lint` to work with no leftover files, and a one-shot purge to remove any old manual install — so the plugin is the single source of truth.

## Scope
**In:** scoped agent refs in skills; `${CLAUDE_PLUGIN_ROOT}`-first preflight discovery in ship/lint; a purge script (`uninstall.{sh,ps1}`, lockstep) + README migration section; apply the purge to both local profiles; ship v1.0.1.
**Out:** new features. Note: scoped agent refs mean the **legacy fallback installer's** agent-dispatch skills no longer fan out (no plugin namespace) — accepted (plugin is primary; [[1.10]] D5).
