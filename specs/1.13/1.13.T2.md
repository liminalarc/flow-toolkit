## Goal
Make `/flow:ship` and `/flow:lint` find the shared helper under a pure-plugin install. `${CLAUDE_PLUGIN_ROOT}` expands in command/skill prompt text (confirmed), and the plugin bundles `hooks/flow-preflight.sh` there.

- In `commands/ship.md` and `commands/lint.md`, prepend `${CLAUDE_PLUGIN_ROOT}/hooks/flow-preflight.sh` to the locate list, keeping the existing profile paths (`$CLAUDE_CONFIG_DIR/hooks`, `~/.claude/hooks`, `~/.claude-*/hooks`) as fallback for the legacy installer.
- Keep the "if none found, stop and tell the user to reinstall" behavior as the final fallback.

## Done when
- [ ] Both commands try `${CLAUDE_PLUGIN_ROOT}/hooks/flow-preflight.sh` first, then the profile paths.
- [ ] Guards (`flow-commit-guard`/`flow-spec-guard`) are unaffected — they already resolve it via `$SCRIPT_DIR` relative to themselves (works under the plugin).

## Notes
The guards call `$SCRIPT_DIR/flow-preflight.sh`; under the plugin `$SCRIPT_DIR` = `${CLAUDE_PLUGIN_ROOT}/hooks`, so they're fine — only the command *prompts* hardcode profile paths.
