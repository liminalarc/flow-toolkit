## Goal
Rewrite every skill's agent-dispatch instruction to use the plugin-scoped name, so dispatch works in a pure-plugin install (no leftover bare agent files to shadow).

- `skills/run/reference/implement.md`: `flow-implementer` → `flow:flow-implementer`, `flow-verifier` → `flow:flow-verifier`.
- `skills/hunt/SKILL.md`: `flow-researcher` → `flow:flow-researcher`.
- `skills/review/SKILL.md`: `flow-reviewer` → `flow:flow-reviewer`.
- `skills/pr/SKILL.md`: `flow-pr-reviewer` → `flow:flow-pr-reviewer`.
- Only the **dispatch target** references change. Agent file names + front-matter `name:` stay `flow-*` (that's the plugin's agent name; the scope prefix is the plugin name). Prose that merely *names* an agent (e.g. "read-only") needn't change, but the "launch/dispatch a <agent>" call sites must be scoped.

## Done when
- [ ] Every "dispatch/launch a <agent>" instruction in run/hunt/review/pr uses `flow:flow-*`.
- [ ] Scoped dispatch confirmed resolvable (smoke: `flow:flow-reviewer` already verified; spot-check one more).
