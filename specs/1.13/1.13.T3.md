## Goal
Give anyone who used the old manual install a one-shot way to remove every artifact and rely solely on the plugin — the repeatable process the maintainer + other users need.

- `uninstall.sh` + `uninstall.ps1` (lockstep), same profile-detection as `install.{sh,ps1}`. For each detected profile, remove the toolkit's manual-install artifacts:
  - `commands/`: `flow*.md` + the renamed `init.md`/`lint.md`/`ship.md` **only if they are the toolkit's** (guard: match content or just the known set) — safer to target the known set (`flow.md flow-hunt.md flow-init.md flow-lint.md flow-ship.md flow-pr.md flow-review.md init.md lint.md ship.md`).
  - `skills/`: `flow flow-hunt flow-review flow-pr run hunt review pr`.
  - `agents/`: `flow-implementer.md flow-verifier.md flow-researcher.md flow-reviewer.md flow-pr-reviewer.md`.
  - `hooks/`: `flow-*.sh` (spec/claude/commit/session + preflight) — plugin bundles its own.
  - Deregister flow hooks from `settings.json` (backup to `.bak`), removing only entries whose command references `flow-*.sh`; preserve all other settings/hooks.
- Print what it removed per profile; never touch non-toolkit files (exact-name/known-set only).
- **Does NOT** run `claude plugin uninstall` (that's the user's choice) and does not remove the plugin — it purges the *manual* install so the plugin stands alone.
- README: a "Migrating off the manual install" section — install plugin → restart → run `uninstall.{sh,ps1}` → restart. Link from the existing migration note.

## Done when
- [ ] `uninstall.sh`/`uninstall.ps1` exist, in lockstep, detecting all profiles; remove the known toolkit artifacts + deregister settings.json flow hooks (backup first); print a per-profile summary; leave non-toolkit files untouched.
- [ ] `install.ps1`-style parse check passes on `uninstall.ps1`.
- [ ] README documents the migration/purge steps.
