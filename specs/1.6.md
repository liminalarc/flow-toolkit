## Problem
Big specs have nowhere to put task-level "how" detail, and the whole spec model assumes a flat `specs/<id>.md`. Spec 1.2 ratified a dual shape: small specs stay flat; big ones earn a `specs/<id>/` directory (orchestrator `<id>.md` + task files `<id>.T<n>.md`). The hooks and commands must support both shapes, and existing flat specs across other flow projects need a migration path.

## Value
As a flow user I want a big spec to break its "how" into task files without breaking any guard or command, and to migrate an existing flat spec into that shape on demand, so that large specs are tractable and the toolkit's own dogfooding path (1.2 → `specs/1.2/`) works.

## Scope
**In:** dual-shape support across `flow-spec-guard.sh`, `flow-preflight.sh`, `flow-commit-guard.sh`, `/flow-lint`, `/flow`; task-file format (local AC, no status; D2); `/flow-lint --migrate` extension to convert flat → dir; README + `hooks.test.sh` coverage.
**Out:** sub-agents/implementers that act on tasks (spec 1.7).

## Acceptance criteria
- [ ] `flow-preflight.sh` `resolved` + `to_resolves` accept dir-form `specs/<id>/<id>.md` and `specs/archive/<id>/<id>.md` — proven by a `hooks.test.sh` case.
- [ ] `flow-spec-guard.sh` validates a task file `<id>.T<n>.md` (id==stem, no status; local-AC nudge) and does not misfire the orchestrator rule — `hooks.test.sh` case.
- [ ] `/flow-lint` treats task files as non-orphans and archives the whole dir; `/flow` `--add`/done-of-done create/move the dir.
- [ ] `/flow-lint --migrate` converts a flat `specs/<id>.md` → `specs/<id>/<id>.md` (dogfooded on 1.2).
- [ ] README documents the dual shape + task files + migration.

## Decisions
- 2026-07-14 — **Task-file detection = stem `<X>.T<n>` AND parent dir named `<X>`** (not stem alone) so a flat `specs/1.T2.md` is never misread as a task and the orchestrator `specs/<id>/<id>.md` never misfires the task rule.
- 2026-07-14 — **Task-file template = `id`/`title` + `## Goal` (the how) + `## Done when` (local AC checkbox — the implementer/verifier seam) + `## Notes`.** No status, no deferrals (D2: those stay on the orchestrator).
- 2026-07-14 — **Local-AC nudge is soft (exit 0), never a block** — matches D1 (breakout unenforced) and the existing bloat-warning pattern; fires when a task file has no `- [ ]`/`- [x]` line.
- 2026-07-14 — **Migration overloads `/flow-lint --migrate <id>`** (flat→dir for one spec) vs. bare `--migrate` (legacy-inline→model); dry-run default, `--apply` writes, git-mv preserves history.
- 2026-07-14 — **Dogfood target = archived 1.2** → `specs/archive/1.2/1.2.md`, exercising the archive dir-form fallback and fulfilling 1.2's own progress-log promise.
- 2026-07-14 — **`flow-commit-guard.sh` needs no edit** — it delegates to preflight and carries no inline `specs/<id>.md` path; slice 1 flows through it (verified by test).
