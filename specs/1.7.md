## Problem
The toolkit runs everything in one context and always pauses for plan approval. Spec 1.2 ratified a sub-agent catalog (isolated context) and a layered autonomy model with a verifier safety net, so routine work can run with less hand-holding without losing control of direction.

## Value
As a flow user I want per-task implementer agents whose diffs an independent verifier checks before integration, and a `checkpoint`/`auto-build` control, so that large specs build in parallel and `auto-build` is tolerable because nothing merges unverified.

## Scope
**In:** implementer + verifier sub-agent definitions; autonomy front-matter (`autonomy: checkpoint|auto-build`, default `checkpoint`); `.flow-toolkit.json` `autonomy.default` + `autonomy.force`; precedence project-force > repo-default > per-spec; verifier gating mode-dependent (D3: blocking auto-build / advisory checkpoint), single verifier, one bounded retry then escalate; wiring into `/flow`'s build loop.
**Out:** reviewer/researcher agents (ship with their skills, 1.8/1.9/1.11); task-file model (1.6).

## Acceptance criteria
- [ ] Implementer + verifier agents defined (purpose, tools, R/W, invocation) per the 1.2 catalog.
- [ ] Autonomy resolved with correct precedence; `auto-build` never bypasses Claude Code permissions or self-approves.
- [ ] Verifier gates per D3: blocking under auto-build, advisory under checkpoint; failing auto-build → 1 retry → escalate.
- [ ] `/flow` build loop dispatches one implementer per task and one verifier before integration.

## Plan (thin slices)

2. [ ] Implementer agent + worktree isolation for parallel tasks.
3. [ ] Autonomy resolution (front-matter + repo default + force) + precedence.
4. [ ] Wire into `/flow` build loop; document.
