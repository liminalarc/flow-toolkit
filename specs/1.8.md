## Problem
`flow-review` is a monolithic command that loads whole and audits four lenses (docs/UX/marketing/product) serially in one context. Spec 1.2 chose it as the first skill migration — its lenses are the cleanest independent, read-only fan-out and the most natural progressive-disclosure split. It's also the repo's **first skill**, so it introduces the skills artifact type + its distribution.

## Value
As a flow user I want `/flow-review` to load only the lens rubric I invoke and to fan the audit out to one reviewer agent per lens with the main thread synthesizing, so that reviews are faster, context stays lean, and the skill + parallel-reviewer pattern is proven for later migrations (1.9/1.11).

## Scope
**In:** convert `flow-review` command → skill (`skills/flow-review/SKILL.md` + one `reference/<lens>.md` rubric per lens, loaded on demand); delete `commands/flow-review.md`; `flow-reviewer` sub-agent (read-only — audits, main thread applies any fixes on confirm); per-lens dispatch (parallel on all-lenses, one on single-lens) + main-thread synthesis; **skills distribution via both installers** (new `skills/` source dir); entry point stays `/flow-review`.
**Out:** `flow-pr` (1.9), `flow`/`flow-hunt` (1.11); plugin packaging (1.10 — folds skills into the plugin later).

## Acceptance criteria
- [ ] `flow-review` is a skill; each lens rubric is a `reference/*` file loaded only on that path — proven by SKILL.md structure + a smoke run.
- [ ] One `flow-reviewer` per lens runs in parallel (read-only); main thread synthesizes a single report and owns any writes.
- [ ] `/flow-review` behaves identically at the entry point after reinstall (skill outranks/replaces the deleted command) — no user-visible regression.
- [ ] Both installers (lockstep) copy `skills/*/` into each profile; dry-run verified.
- [ ] README + CLAUDE.md docs updated (skill as a product part, commands list, install steps).

Matches the 1.2 catalog, keeps parallel agents from concurrent writes, preserves "confirm before significant doc changes." Refines today's `--docs` "Update" phase into findings→confirmed-apply.
